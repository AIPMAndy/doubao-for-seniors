# 豆包使用网页 + 开源发布（示例）

## 输入背景示例

- 目标：做一个“豆包使用大全”的网页版本并开源。
- 已有资产：多端文档（Web/iOS/Android/桌面/API）已整理。
- 约束：中文优先、信息可靠、可持续更新。

## 乔布斯式拆解

### 1) Mission

让普通用户在 5 分钟内找到自己设备上的豆包最佳使用路径。

### 2) Primary User

- 新手用户：首次接触豆包，需要快速上手。
- 进阶用户：需要 API 与实战效率路径。

### 3) Product Thesis

相比零散教程与搜索结果，这个网页用统一结构和真实场景，
帮助用户以最短路径学会豆包并可直接复用到工作。

### 4) MVP 建议

保留：
- 首页价值说明 + 端侧入口导航
- 五端核心能力对比
- 新手 3 分钟上手区
- 开发者 API 快速开始
- 开源贡献入口

延后：
- 多语言版本
- 自动化截图采集
- 用户账号系统

### 5) 网页信息架构建议

1. Hero：一句话价值 + CTA（立即开始）
2. 平台导航：Web / iOS / Android / 桌面 / API
3. 能力矩阵：可视化对比
4. 场景教程：学习 / 办公 / 创作 / 开发
5. 开源区：仓库地址、贡献方式、更新日志

### 6) 开源发布包建议

- `README.md`：项目定位、预览、快速启动
- `docs/`：延续现有文档结构
- `assets/screenshots/`：统一命名和占位说明
- `CONTRIBUTING.md`：PR 规范与核验要求
- `ROADMAP.md`：下一阶段规划

### 7) 首周反馈闭环

- 指标：页面停留时长、文档点击率、Star 增长、Issue 数量
- 动作：按 Issue 标签每周迭代一次内容结构
