# Web 端使用指南（增强版）

> 核验入口：`https://www.doubao.com/`  
> 内容定位：让新手 3 分钟上手，让进阶用户 30 分钟形成稳定工作流。

## 1. 适合谁

- 经常在浏览器中处理学习、办公、写作、研究、代码任务的用户。
- 需要“快速问答 + 深度任务拆解 + 多轮迭代”能力的用户。
- 不想安装客户端，希望直接在线使用 AI 的用户。

## 2. 3 分钟快速上手

1. 打开 `https://www.doubao.com/`，登录账号。
2. 新建对话，先发一条“任务定义句”：

```text
目标：我需要在今天下班前产出一份可直接发给老板的周报。
背景：我是运营，负责新媒体投放。
约束：600 字以内，分 4 段，语气专业。
输出：先给提纲，再给正文。
```

3. 收到首版后继续追问：

```text
请按“可执行性”重写：每段都给一个可量化指标。
```

## 3. 提问框架（建议长期复用）

建议统一使用四段式：`背景 -> 目标 -> 约束 -> 输出格式`

```text
背景：我在做 XX，当前遇到 XX。
目标：希望得到 XX 结果。
约束：时间/篇幅/语气/禁用项。
输出：请用清单/表格/模板形式输出。
```

## 4. 五大高频工作流

### 4.1 学习提效

```text
请把这段内容拆成：
1) 必懂概念 2) 易错点 3) 练习题（含答案）
```

### 4.2 写作提效

```text
主题：[你的主题]
读者：[目标人群]
风格：[专业/口语/故事化]
请先给 3 个标题，再输出 1200 字正文。
```

### 4.3 信息总结

```text
请把这份材料做成“领导速读版”：
- 5 条结论
- 3 个风险
- 3 个行动建议
```

### 4.4 会议纪要

```text
请将以下会议记录整理为：
1) 决策事项 2) 待办清单（负责人+截止时间）3) 风险点。
```

### 4.5 代码辅助

```text
请先给最小可运行示例，再解释每一段代码的作用，最后列出 3 个常见报错与修复方式。
```

## 4.6 Web 端 Prompt 加油包（可直接复制）

### A. 写作类（4 条）

```text
1) 请按“标题-开头-正文-结尾CTA”输出一篇 800 字文章，读者是[人群]。
2) 请给我 5 个标题，分别是：专业版、口语版、故事版、数据版、争议版。
3) 把这段内容改写成“领导30秒读完版”，最多5条。
4) 帮我写一封工作邮件：背景、问题、建议、下一步。
```

### B. 办公类（4 条）

```text
1) 请把这段会议记录整理成：决策事项/待办人/截止时间。
2) 请把这个方案拆成“今天就能执行”的 6 个动作。
3) 输出一个周报模板：本周成果/风险/下周计划/需协助事项。
4) 请把这份材料做成 1 页汇报提纲，结构为“问题-分析-建议-收益”。
```

### C. 研究与总结类（4 条）

```text
1) 请先给结论，再给依据，最后给风险点。
2) 把问题拆成3个子问题分别回答，最后合并总论。
3) 请给我“支持观点”和“反对观点”各3条。
4) 请输出“可验证信息”和“待核验信息”两栏表格。
```

### D. 追问增强类（4 条）

```text
1) 请把答案改成表格，并加上优先级。
2) 请再给一个更保守、更稳妥的版本。
3) 请用小学生也能懂的话再解释一遍。
4) 请把最终结果缩成“10秒读完版 + 60秒读完版”。
```

## 5. 质量校验清单（交付前必看）

- 结论是否对应你的原始目标？
- 关键事实、价格、政策、日期是否人工核验？
- 输出格式是否可直接复制到邮件/文档/PPT？
- 是否需要“再精简一版”和“更详细一版”各一份？

## 6. 常见问题与解决

### Q1：回答太泛，没法直接用？

补充 3 类约束：
- 角色约束：你是[岗位/行业]专家
- 结果约束：必须包含[表格/步骤/指标]
- 反例约束：不要空话，不要泛泛建议

### Q2：长对话后模型“跑偏”？

- 开新会话并粘贴“核心上下文摘要”。
- 每 4-6 轮做一次“目标重申”。

### Q3：担心隐私？

- 不上传身份证号、银行卡号、客户合同原文等敏感内容。
- 用占位符替代：`客户A`、`项目X`、`金额Y`。

## 7. 网页内容结构建议（用于你的网站）

将 Web 指南在网页中拆成以下模块：

1. 首屏价值：一句话说明“Web 端能解决什么问题”。
2. 3 分钟上手：图文步骤。
3. 高价值提示词库：可复制按钮。
4. 五大工作流：按学习/办公/写作/研究/代码分区。
5. 常见问题：折叠面板。
