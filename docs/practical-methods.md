# 豆包具体使用方法手册（场景化实操）

> 目标：不依赖固定截图，只靠“步骤 + Prompt + 追问 + 验收”就能持续可用。

## 1. 通用操作框架（先记住）

每个任务都按这 4 步走：

1. **说清任务**：你要做什么。
2. **加上约束**：字数、语气、时间、格式。
3. **先小后大**：先要简版，再要详细版。
4. **追问优化**：不满意就改，不要一次问完。

推荐万能模板：

```text
我现在要做：[任务]
请按以下要求输出：[约束]
先给我简版，再给我详细版。
最后给“今天就能执行”的3步。
```

## 2.1 天气出门法

- 场景：出门前想快速判断穿衣和带伞。
- 操作步骤：问天气 → 问穿衣 → 问风险。

```text
请告诉我今天[城市]天气，并给我穿衣建议。
最后补充：是否需要带伞、是否要注意大风或降温。
```

追问：`请给我“老人外出注意事项”3条。`

验收标准：看完 15 秒内能做出“穿什么/带不带伞”决定。

<a id="method-02"></a>
## 2.2 医院就诊法

- 场景：第二天去医院，不清楚路线和准备事项。
- 操作步骤：给地点 → 要路线 → 要清单。

```text
我明天去[医院名称]，请给我地铁和打车两种路线。
再给我“出门前准备清单”，按优先级排序。
```

追问：`请把清单改成“我可以直接照着做”的短句版。`

验收标准：路线 + 准备物品 + 出发时间建议齐全。

<a id="method-03"></a>
## 2.3 药盒识别法

- 场景：看不懂药盒或说明书。
- 操作步骤：上传图片 → 要识别文字 → 要重点总结。

```text
请先识别图片中的文字，再告诉我：
1) 作用
2) 常见用法
3) 注意事项
请用通俗中文。
```

追问：`请把“注意事项”改成家人提醒语气。`

验收标准：能看懂“用途/用法/风险”，且不会误解复杂术语。

<a id="method-04"></a>
## 2.4 体检解读法

- 场景：体检报告有异常项，不知道如何理解。
- 操作步骤：贴报告文字 → 要分条解释 → 要行动建议。

```text
请解释这段体检结果，按“是什么-意味着什么-下一步建议”输出。
不要使用专业术语。
```

追问：`请标出“必须问医生”的项目。`

验收标准：能区分“可观察”与“需就医”，不被误导。

<a id="method-05"></a>
## 2.5 家庭通知法

- 场景：要在家庭群发通知。
- 操作步骤：给对象 + 给语气 + 给字数。

```text
帮我写一条家庭群通知，主题是[主题]。
要求：礼貌、简短、重点在前，60字以内。
```

追问：`再给我一个更正式版和一个更亲切版。`

验收标准：复制即发，不需二次改写。

<a id="method-06"></a>
## 2.6 节日祝福法

- 场景：生日、节日、纪念日祝福。
- 操作步骤：给对象 + 给风格 + 要多版本。

```text
帮我写3条[节日/生日]祝福，送给[对象]。
要求：真诚、不俗套，每条50字以内。
```

追问：`第2条改成更幽默一点。`

验收标准：至少 1 条达到“直接可发”。

<a id="method-07"></a>
## 2.7 家常做饭法

- 场景：不知道做什么菜。
- 操作步骤：说食材 → 说口味 → 要步骤。

```text
我有[食材]，请给我3道菜。
每道菜输出：食材用量、4步做法、注意事项。
```

追问：`请改成少油少盐版本。`

验收标准：步骤清晰，按步骤可直接做。

<a id="method-08"></a>
## 2.8 旅行规划法

- 场景：自由行计划。
- 操作步骤：说天数 + 预算 + 人群特点。

```text
请帮我做[城市]3天旅行计划。
要求：少走路、预算[金额]、每天给交通和餐食建议，表格输出。
```

追问：`再给我一个“雨天替代方案”。`

验收标准：计划可执行，有预算和替代路线。

<a id="method-09"></a>
## 2.9 防诈骗核查法

- 场景：收到可疑消息。
- 操作步骤：贴消息 → 判风险 → 给核查顺序。

```text
请判断这条消息是否存在诈骗风险。
请给我5条核查步骤，按先后顺序。
```

追问：`请加上“绝对不要做的3件事”。`

验收标准：能明确“先核实再行动”，避免冲动转账。

<a id="method-10"></a>
## 2.10 家庭教学法

- 场景：子女教父母使用豆包。
- 操作步骤：给原文 → 转成长辈可读版 → 转成口语讲解稿。

```text
请把这段内容改写成适合60岁长辈阅读的版本：
短句、少术语、重点前置。
```

追问：`请再生成“我可以读给爸妈听”的口语稿。`

验收标准：家人读一遍即可理解并开始操作。

<a id="method-11"></a>
## 2.11 学习计划法

- 场景：想持续练习，但怕坚持不住。
- 操作步骤：定周期 → 定时长 → 要每日动作。

```text
请给我7天豆包学习计划，每天10分钟。
每天包含：1个目标、1个练习、1个自测题。
```

追问：`请把计划改成“周一到周日打卡表”。`

验收标准：每天任务清晰，能打卡复盘。

<a id="method-12"></a>
## 2.12 每日复盘法

- 场景：想持续提升回答质量。
- 操作步骤：回顾任务 → 总结得失 → 定明日动作。

```text
请帮我做今天豆包使用复盘：
1) 我做对了什么
2) 我卡在哪
3) 明天3个可执行动作
```

追问：`请把明天动作按“最容易开始”排序。`

验收标准：第二天能直接按清单行动。

<a id="method-13"></a>
## 2.13 听不懂重述法

- 场景：回答太快或太复杂，没听懂。
- 操作步骤：先要求重述 → 再要求分步 → 最后要例子。

```text
我没听懂，请你用更简单的话再说一遍，分3步讲，并举1个生活例子。
```

追问：`请把每一步控制在15个字以内。`

验收标准：能复述出“第一步做什么、第二步做什么”。

<a id="method-14"></a>
## 2.14 电话沟通提词法

- 场景：要打电话沟通事情，怕说不清。
- 操作步骤：说明对象 → 说明目的 → 生成开场与收尾。

```text
我要给[对象]打电话沟通[事情]，请给我：
1) 开场白 2) 核心3句话 3) 结束语。
```

追问：`请再给一个“更礼貌但更坚定”的版本。`

验收标准：照稿读即可完成沟通，不跑题。

<a id="method-15"></a>
## 2.15 家庭群沟通法

- 场景：家庭群容易误会，想表达更温和。
- 操作步骤：给原话 → 指定语气 → 生成三版。

```text
请把这段话改成家庭群可读版本：礼貌、简短、避免冲突。
给我温和版/中性版/正式版3个版本。
```

追问：`请把温和版控制在50字内。`

验收标准：信息明确、情绪稳定、减少误解。

<a id="method-16"></a>
## 2.16 政务办事清单法

- 场景：办理社保、公积金、证件业务。
- 操作步骤：说办事项 → 要材料清单 → 要流程顺序。

```text
我要办理[事项]，请给我：
1) 需要准备的材料
2) 办理流程
3) 容易遗漏的点
```

追问：`请按“出门前-现场-回家后”三个阶段整理。`

验收标准：按清单走，现场不漏资料。

<a id="method-17"></a>
## 2.17 快递异常处理法

- 场景：快递延误、丢件、破损。
- 操作步骤：描述问题 → 要维权话术 → 要处理路径。

```text
我的快递出现[问题]，请给我客服沟通话术和处理步骤。
```

追问：`请给我“先做什么再做什么”的时间线版本。`

验收标准：能快速进入售后流程并保留证据。

<a id="method-18"></a>
## 2.18 银行业务咨询法

- 场景：银行卡、转账、理财说明看不懂。
- 操作步骤：贴原文 → 要通俗解释 → 要风险提醒。

```text
请把这段银行说明翻成通俗中文，并告诉我风险点。
```

追问：`请标出“我必须去柜台确认”的部分。`

验收标准：明确“能线上办”与“必须线下核实”。

<a id="method-19"></a>
## 2.19 网购比价决策法

- 场景：同类商品太多，不知道选哪个。
- 操作步骤：给候选项 → 给预算 → 要对比表。

```text
请比较这3个商品：价格、优缺点、适合人群，并给推荐顺序。
```

追问：`请给我“最省钱”和“最省心”两个结论。`

验收标准：10分钟内完成下单决策。

<a id="method-20"></a>
## 2.20 售后投诉法

- 场景：商品或服务不满意，需要投诉。
- 操作步骤：写事实 → 写诉求 → 生成正式投诉文本。

```text
请根据以下事实，帮我写一段投诉说明：
要求客观、清晰、可维权。
```

追问：`请再给“强硬版”和“协商版”各一份。`

验收标准：投诉信息完整，诉求明确。

<a id="method-21"></a>
## 2.21 孩子辅导法

- 场景：想辅导孩子学习，但不知从何入手。
- 操作步骤：给年级 + 科目 + 当前问题。

```text
请给[年级][科目]设计10分钟辅导计划：讲解-练习-检查。
```

追问：`请补一个“家长不会讲时也能照着做”的版本。`

验收标准：孩子能当场完成1个小练习。

<a id="method-22"></a>
## 2.22 作业讲解法

- 场景：看不懂题目解析。
- 操作步骤：贴题目 → 要分步讲解 → 要同类题。

```text
请分步骤讲这道题：先讲思路，再讲做法，最后给1道同类题。
```

追问：`请把讲解改成“小学生能懂”的版本。`

验收标准：能说出“为什么这样做”，不是只记答案。

<a id="method-23"></a>
## 2.23 英语口语练习法

- 场景：出行或日常想练简单英语。
- 操作步骤：给场景 → 要中英对照 → 要发音提示。

```text
请给我“问路/点餐/购物”各3句英语，附中文和慢速读法。
```

追问：`请把每句控制在8个词以内。`

验收标准：能开口说出至少3句完整句子。

<a id="method-24"></a>
## 2.24 短视频脚本法

- 场景：想拍家庭生活短视频。
- 操作步骤：给主题 → 要脚本结构 → 要台词。

```text
请写一个60秒短视频脚本：开场3秒、主体40秒、结尾CTA。
```

追问：`请改成“普通话口播更顺口”的版本。`

验收标准：直接按稿拍，不需要大量重写。

<a id="method-25"></a>
## 2.25 一周菜谱法

- 场景：每天纠结吃什么。
- 操作步骤：给人数和口味 → 要周计划 → 要购物清单。

```text
请给我一周晚餐计划（2人份），清淡口味，并生成购物清单。
```

追问：`请把周三改成更快手的20分钟菜谱。`

验收标准：减少每天决策压力，能直接采购。

<a id="method-26"></a>
## 2.26 慢病记录法

- 场景：需要长期记录血压/血糖等指标。
- 操作步骤：给指标 → 要记录模板 → 要提醒规则。

```text
请给我血压记录模板：日期、早晚数值、备注、异常提醒。
```

追问：`请加“什么情况要尽快就医”的提示。`

验收标准：记录格式稳定、异常可及时发现。

<a id="method-27"></a>
## 2.27 情绪疏导法

- 场景：心情低落、焦虑、睡前胡思乱想。
- 操作步骤：描述状态 → 要安抚建议 → 要行动清单。

```text
我现在有点焦虑，请给我3个可以马上做的放松动作。
```

追问：`请给一个睡前5分钟版本。`

验收标准：情绪强度下降，能恢复到可行动状态。

<a id="method-28"></a>
## 2.28 纪念日管理法

- 场景：怕忘记家人生日或重要日子。
- 操作步骤：输入日期清单 → 要提醒计划 → 要祝福模板。

```text
请根据这些日期，给我全年提醒计划，并附每月祝福模板。
```

追问：`请把提醒改成“提前7天/提前1天”双提醒。`

验收标准：全年关键日期不遗漏。

<a id="method-29"></a>
## 2.29 文件阅读法

- 场景：收到长文件看不完。
- 操作步骤：贴文本 → 要速读版 → 要行动版。

```text
请把这份文件做成“30秒速读版 + 行动清单版”。
```

追问：`请标出最容易忽略但最重要的2条。`

验收标准：短时间抓住重点并明确下一步。

<a id="method-30"></a>
## 2.30 信息真伪核验法

- 场景：看到热点消息，不确定真假。
- 操作步骤：贴消息 → 要核验路径 → 要结论等级。

```text
请给我这条消息的核验步骤：先查来源、再查日期、再查官方渠道。
并给“可信/存疑/高风险”判断。
```

追问：`请告诉我最权威的3个核验渠道。`

验收标准：不轻信、不转发未核实信息。

## 4. 追问模板（回答不理想时）

```text
1) 请再简化，给小白版本。
2) 请给一个更稳妥、风险更低的版本。
3) 请用表格输出，并按优先级排序。
4) 请指出你刚才回答里最不确定的部分。
5) 请给一个“今天就能开始”的最小行动版本。
```

## 5. 安全边界（固定加一句）

高风险场景统一追加：

```text
以上仅供参考，涉及医疗/法律/投资问题请咨询专业人士。
```
